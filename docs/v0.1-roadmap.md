# homework-ai v0.1 产品路线图（MVP -> 可商用）

## 1. 目标与定位

`homework-ai` 聚焦中国小学生课后学习，提供「课时规划 + 自适应作业 + 题目辅导」的一体化体验。  
v0.1 的核心目标不是“大而全”，而是先把“真实可用的学习闭环”跑通。

---

## 2. 阶段划分

## 阶段A：MVP（当前 + 2~4周）

**目标**：单学生可用，完成最小学习闭环。  

### 功能范围
- 课时级预习/学习/复习/作业自动生成
- 自适应练习（答对升难、答错降难）
- 错题库沉淀与重练
- 学生自主提问（图文输入）与分步辅导
- 基础用户系统（注册/登录）

### 技术范围
- 前端：静态页面 + 交互逻辑
- 后端：Express API + JSON 存储（可替换 DB）
- AI：OCR/LLM 占位接口（mock + 可替换真实服务）

### 验收标准
- 学生能连续完成一节课完整学习流程
- 错题能自动记录、展示、复练
- 后端有可调用 API，关键数据可持久化

---

## 阶段B：Beta（1~2个月）

**目标**：小范围班级试点，能被老师和家长“看见价值”。  

### 新增功能
- 教师端：布置作业、查看学情、错题分布
- 家长端：学习报告、完成率、复习提醒
- 题目标签体系：知识点/难度/题型
- 课堂与课后联动：按教材版本匹配课时

### 新增工程能力
- 数据库升级（PostgreSQL/MySQL）
- 对象存储（图片题）
- 任务队列（OCR/批改异步）
- 基础日志和告警

### 关键指标
- 7日留存 > 25%
- 作业完成率提升 > 20%
- 错题二次正确率提升 > 15%

---

## 阶段C：可商用（2~4个月）

**目标**：支持学校/机构稳定使用。  

### 商业化能力
- 多租户（学校/机构隔离）
- 权限体系（学生/家长/老师/教务）
- 计费策略（按学生数、调用量、套餐）
- 服务等级（SLA）和运维工具

### 合规与安全
- 未成年人信息保护流程
- 数据分级和脱敏
- 审计日志与操作留痕
- 第三方模型调用合规评估

### 运维与交付
- CI/CD 流程
- 自动化测试和回归脚本
- 灰度发布与回滚机制

---

## 3. v0.1 任务清单（建议）

1. 完成后端 API 与前端联调（当前进行中）
2. 接入真实 OCR（拍照识题）
3. 接入真实 LLM（分步提示、错因解释）
4. 增加最小用户体系（学生账号）
5. 输出学习记录日报（个人维度）

---

## 4. 风险与应对

1. **内容版权风险**
   - 应对：采用官方来源目录 + 授权接入，不做未授权批量抓取分发。

2. **模型幻觉风险**
   - 应对：分步提示优先、不直接给答案；敏感输出加规则校验。

3. **未成年人数据风险**
   - 应对：最小化采集、明示用途、可删除、可审计。

4. **成本波动风险（OCR/LLM）**
   - 应对：分层模型策略（轻量优先 + 高频缓存 + 限流）。
